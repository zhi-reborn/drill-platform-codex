## 1. 项目概述

### 1.1 项目背景

构建一套面向 IT 生产系统的故障演练流程管理平台，核心解决"演练过程不可见、多方协同难管控、演练结果难追溯"的问题。系统以**指挥中心大屏**为核心展示载体，以**流程引擎**为驱动核心，实现演练全流程的可视化编排、实时协同与数字化归档。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 流程可视 | 大屏实时展示演练进度，步骤状态秒级同步 |
| 协同可控 | 各角色按流程节点操作，完成即确认，超时即预警 |
| 过程可溯 | 完整记录演练时间线、操作人、耗时、备注，自动生成报告 |
| 快速复用 | 支持演练模板化，一次编排多次复用 |

### 1.3 设计原则

- **轻量优先**：MVP 阶段不引入重型工作流引擎，自研轻量级状态机引擎
- **实时优先**：WebSocket 全双工通信，状态变更端到端延迟 < 1s
- **扩展优先**：模块化设计，后续可无缝接入真实监控指标、故障注入能力

### 3.2 自研流程引擎设计

Go 生态缺乏类似 Java Flowable 的成熟引擎，本系统采用**事件驱动状态机（Event-Driven FSM）**自研轻量引擎。

#### 3.2.1 核心概念

| 概念 | 说明 |
|------|------|
| FlowDef | 流程定义（模板），描述步骤、依赖、超时、执行人 |
| FlowInst | 流程实例（一次演练），记录当前状态、开始/结束时间 |
| StepDef | 步骤定义，包含：ID、名称、类型、超时、前置步骤、执行人 |
| StepInst | 步骤实例，记录：状态、开始时间、结束时间、操作人、备注 |
| Event | 事件：StepComplete、StepTimeout、DrillPause、DrillTerminate |
| State | 状态：Pending → Running → Completed / Timeout / Skipped |

#### 3.2.2 步骤类型

```go
type StepType string

const (
    StepTypeSerial   StepType = "serial"    // 串行：上一步完成自动激活
    StepTypeParallel StepType = "parallel"  // 并行：多分支同时激活，全部完成才继续
    StepTypeAnyOf    StepType = "any_of"    // 或签：多分支中任意一个完成即可继续
    StepTypeCondition StepType = "condition" // 条件分支：根据上一步结果走不同分支
)
```

#### 3.2.3 引擎核心接口

```go
// internal/pkg/flowengine/engine.go
type Engine interface {
    // 创建流程实例
    CreateInstance(flowDefID int64, assignees map[int64][]int64) (*FlowInst, error)

    // 启动演练
    Start(instanceID int64) error

    // 完成步骤
    CompleteStep(instanceID, stepID int64, operatorID int64, remark string) error

    // 上报问题（步骤阻塞）
    ReportIssue(instanceID, stepID int64, operatorID int64, issue string) error

    // 指挥员干预
    Intervene(instanceID int64, action InterveneAction, targetStepID *int64) error

    // 查询实例状态
    GetInstance(instanceID int64) (*FlowInst, error)
}

type InterveneAction string
const (
    ActionPause     InterveneAction = "pause"      // 暂停
    ActionResume    InterveneAction = "resume"     // 恢复
    ActionTerminate InterveneAction = "terminate"  // 终止
    ActionSkip      InterveneAction = "skip"       // 跳过某步
    ActionForceComplete InterveneAction = "force_complete" // 强制完成
)
```

#### 3.2.4 超时检测机制

```go
// internal/pkg/flowengine/scheduler.go
type TimeoutScheduler struct {
    redis       *redis.Client
    eventBus    chan Event
}

// 步骤激活时，写入 Redis ZSET：score = 超时时间戳
// 独立 Goroutine 每秒扫描 ZSET，发现超时的步骤触发 Timeout 事件
func (s *TimeoutScheduler) Scan() {
    for {
        now := time.Now().Unix()
        // ZRANGEBYSCORE timeouts 0 now
        expired := s.redis.ZRangeByScore(ctx, "drill:timeouts", &redis.ZRangeBy{
            Min: "0", Max: strconv.FormatInt(now, 10),
        }).Val()

        for _, key := range expired {
            // 解析 instanceID_stepID
            // 发送 Timeout 事件到 EventBus
            s.eventBus <- Event{Type: EventStepTimeout, ...}
            s.redis.ZRem(ctx, "drill:timeouts", key)
        }
        time.Sleep(1 * time.Second)
    }
}
```

#### 6.2.3 WebSocket 数据绑定

```typescript
// src/stores/websocket.ts
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useWebSocketStore = defineStore('websocket', () => {
    const drillState = ref<DrillState | null>(null)
    const messages = ref<Message[]>([])
    let ws: WebSocket | null = null

    function connect(drillId: number) {
        ws = new WebSocket(`wss://api.example.com/ws/display/${drillId}`)
        ws.onmessage = (event) => {
            const msg = JSON.parse(event.data)
            if (msg.event_type === 'step_complete') {
                updateStepStatus(msg.payload)
            }
            messages.value.unshift(msg)
            if (messages.value.length > 50) messages.value.pop()
        }
    }

    return { drillState, messages, connect }
})
```

## 8. 安全设计

### 8.1 认证与授权

| 层级 | 方案 |
|------|------|
| 认证 | JWT（Access Token + Refresh Token），有效期 2h / 7d |
| 密码 | bcrypt 哈希存储，成本因子 12 |
| 权限 | RBAC：admin / director / executor / viewer |
| 接口鉴权 | Gin Middleware 校验 JWT + 角色权限 |

### 8.2 数据安全

- 敏感操作（开始/终止/强制完成）需二次确认，并记录审计日志
- 演练报告导出支持水印
- 数据库连接使用 SSL（生产环境）

### 8.3 传输安全

- 全站 HTTPS（WSS 对应 WebSocket）
- API 防重放：请求头携带 `X-Request-ID` + 时间戳校验
