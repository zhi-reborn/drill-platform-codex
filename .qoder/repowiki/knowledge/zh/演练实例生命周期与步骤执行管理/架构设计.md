- **分层架构**：遵循 `Handler` (API层) -> `Service` (业务逻辑层) -> `Repository` (数据访问层) -> `Entity` (领域实体) 的经典分层结构。
- **核心组件**：
  - `internal/api/handler/drill/handler.go`：定义 RESTful 接口，处理 HTTP 请求参数绑定与响应格式化，集成 `middleware` 获取用户上下文。
  - `internal/service/drill.go`：核心业务逻辑，协调 `DrillRepo`、`TemplateRepo`、`StepRepo` 进行数据操作，并集成 `flowengine`（流程引擎）驱动演练状态机。
  - `internal/repository/drill.go` 与 `step.go`：基于 GORM 实现数据持久化，包含复杂的关联查询（Preload）和事务控制。
  - `internal/domain/entity/instance.go`：定义 `DrillInstance` 和 `StepInstance` 核心模型，映射数据库表结构。
- **依赖方向**：Handler 依赖 Service，Service 依赖 Repository 和外部引擎（flowengine），Repository 直接操作 DB。
