- 统一响应封装：所有 Handler 方法均使用 `response` 包（如 `response.Success`, `response.BadRequest`）标准化 API 返回格式。
- 步骤目标解析模式：在步骤操作接口中，通过 `resolveStepOperationTarget` 统一处理步骤 ID 到模板 ID 的映射与兜底回填逻辑，确保业务一致性。
- 缓存失效策略：在更新步骤信息（`UpdateStepInfo`）后，显式调用 `InvalidateStepCache` 清除 Redis 中的步骤列表缓存，保证数据实时性。
- 操作日志审计：在演练状态变更（启动/暂停/终止）及关键步骤操作时，同步创建 `DrillInstanceLog` 记录操作人、时间及内容，用于审计追踪。
